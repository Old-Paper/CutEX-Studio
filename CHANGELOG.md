# Changelog

## 0.7.0 - 2026-07-18

- 产品正式更名为 **CutEX Studio**，同步更新工作台品牌、窗口标题、项目对话框、导出日志和 Windows 文件名。
- 新项目默认保存为 `.cutex.json`；旧版 `.dubcut.json` 项目仍可直接打开、原路径保存和加入最近项目。
- 继续沿用旧版本地数据目录，使更名升级后最近项目记录与日志不会丢失。

## 0.6.1 - 2026-07-17

- 自动粗剪全局忽略持续时间小于或等于 1 秒的静音，刚好 1.000 秒也不会生成删除标记。
- 前后保留帧应用后若实际删除区间不超过 1 秒，同样不放入时间轴，减少密集的无意义小标记。
- 快、中、慢三档预设的最短静音统一为 1000 ms，旧项目打开时也会清理不超过 1 秒的自动标记。
- 手工通过快捷键创建的小片段保持不变，避免覆盖用户审听后的编辑决定。

## 0.6.0 - 2026-07-17

- 新增默认“快速导出”：使用 FFmpeg concat 分段清单串行读取保留区间，不再为每个切点创建一组视频/音频滤镜。
- 保留“高精度兼容”模式用于特殊编码和最终母版；快速模式失败时自动回退，无需重新选择文件。
- 快速模式按音视频帧边界裁切，并把音频补齐到预期成片时长，避免大量片段累计音画偏移。
- GPU 加速可与快速模式组合使用，任务进度会明确显示“GPU 正在快速分段导出”。
- 从实际日志确认旧任务包含 762 路视频与 762 路音频滤镜，NVENC 仍只有 `0.16x` 实时速度。
- 使用同一条 49 分钟源文件进行 762 切点压力测试：输出 76.2 秒成片耗时 39.6 秒，达到 `1.95x`，约为旧架构的 12 倍；音视频时长均为 76.200 秒。

## 0.5.0 - 2026-07-17

- 导出窗口增加 CPU、NVIDIA NVENC、Intel Quick Sync、AMD AMF 编码方式选择，CPU 仍为默认稳定模式。
- 打开导出窗口时会用 256×256 测试帧真实初始化每个硬件编码器，只允许选择实际可用的 GPU。
- MP4、MOV、MKV 支持 H.264 GPU 加速；WebM 继续使用 VP9 多线程软件编码，纯音频使用对应音频编码器。
- GPU 在正式素材上初始化失败时自动回退 x264 CPU，不丢失当前导出任务；取消导出不会触发回退重试。
- 当前设备实测 NVIDIA NVENC 与 AMD AMF 可用，Intel Quick Sync 不可用并正确禁用。
- 使用真实 MP4 验证 NVIDIA GPU 成片，以及选择不可用 Intel 时自动回退 CPU 成片。

## 0.4.0 - 2026-07-17

- MP4/MOV/MKV 改用稳定的 x264 `veryfast` 软件编码预设，在保持精确重编码和兼容性的同时明显降低导出等待时间。
- WebM 启用 VP9 多线程行编码并提高稳定速度档位；不默认启用依赖显卡型号和驱动的硬件编码。
- 时间轴支持普通滚轮横向浏览，`Alt + 滚轮` 以鼠标所在时间点为中心放大或缩小。
- 增加全片导航条：点击或拖动快速跳转，放大后可拖动紫色可视窗口快速平移长时间轴。
- 修复低缩放下删除区间被强制扩为 4 像素和最少 0.15% 宽度造成的红色标记假重叠；现在按真实时间比例绘制，最小仅 1 像素。
- 使用真实桌面窗口验证 250% 缩放、滚轮横移 500 像素、全片导航到 80% 位置和标记像素精度。

## 0.3.1 - 2026-07-17

- 修复粗剪片段较多时，FFmpeg 参数超过 Windows 命令行长度限制并报 `spawn ENAMETOOLONG` 的问题。
- 复杂裁剪滤镜改为 UTF-8 临时脚本传递，不再把数万字符直接放进进程启动参数。
- 临时滤镜脚本在导出成功、失败或取消后都会清理。
- 使用真实 MP4 完成 600 个保留片段的 MP3 导出，并验证临时文件无残留。

## 0.3.0 - 2026-07-17

- 选择或拖入媒体后自动使用当前节奏预设开始静音检测并直接生成粗剪，无需再次点击分析。
- 新增 `X` 审听快捷键：播放头在建议删除片段内时恢复该片段，在保留片段内时把相邻切点之间的完整片段标记删除。
- 播放中按 `X` 标记删除后立即跳到该片段末尾，可一边从头审听一边修正粗剪结果。
- 新素材替换进行中的分析时会取消旧任务，并阻止旧结果覆盖当前时间轴。
- 新增播放头片段边界与 `X` 映射测试，并用真实“有声—静音—有声”MP4 验证自动粗剪和播放中反选。

## 0.2.0 - 2026-07-17

- 支持把本地视频或音频直接拖到整个应用窗口导入，并增加拖放遮罩和格式校验。
- 导出前可选 MP4、MOV、MKV、WebM、MP3、WAV、M4A；支持从视频单独提取音频。
- 增加快、中、慢三档节奏预设；中档前后保留 4 帧，慢档保留 8 帧，按素材真实帧率换算。
- 使用 FFmpeg 从所选音轨解码 4 kHz 单声道 PCM，计算 2,400 桶峰值和 RMS，以 Canvas 绘制真实波形。
- 使用真实“声音-静音-声音”MP4 验证波形，并成功生成全部 7 种导出格式。
- 45 个常规测试和 2 个真实 FFmpeg 集成测试通过。

## 2026-07-17

- 发布 0.1.1 修复版。
- 本地媒体协议支持标准 HTTP Range、`206 Partial Content` 和正确 MIME，使视频可可靠跳转而不会回到开头。
- 粗剪预览在播放时自动跳过所有启用删除的静音区间，连续区间会一次跳过，末尾全部删除时停止播放。
- 删除区间按钮改为明确动作：已删除显示“恢复”，已恢复显示“删除”。
- 使用真实 6 秒 MP4 验证：Range 返回 206，画面从开头成功跳至第 4 秒。
- 修复时间轴拖动后播放头被播放器旧 `timeupdate` 回调重置到 0 的问题。
- 播放头与时间轴空白区域现在都支持按住鼠标连续拖动定位。
- 拖动期间暂停媒体进度回写，播放器到达目标位置后再恢复同步。
- 修复删除区间边缘拖动结束后，点击事件冒泡导致播放头跳到区间起点的问题。
- 新增播放同步竞态测试；当前 31 个测试全部通过。
